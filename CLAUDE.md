# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

SprachLichtung ("Language Clearing") is a Docusaurus v3 static educational site focused on critical thinking (*Kritisches Denken*) and language/society topics (*Sprach-Welten*). German is the default locale; English is the secondary locale. The site is hosted at https://sprachlichtung.org and deployed to GitHub Pages.

## Commands

```bash
yarn start              # Dev server (German, default locale)
yarn start:en           # Dev server (English locale)
yarn build              # Build static site to /build
yarn typecheck          # TypeScript type check
yarn clear              # Clear Docusaurus cache (rm -rf .docusaurus build)
yarn serve              # Serve the built /build directory locally
yarn deploy             # Deploy to GitHub Pages
yarn sync:notion        # Sync markdown content with Notion database
yarn write-translations:en  # Generate English translation keys into i18n/en/
```

## Architecture

### Content Structure

Documentation lives under `docs/` in two main trees:
- `docs/kritisches-denken/` — 11 numbered topic modules (010–150) on logic, argumentation, cognitive biases, fallacies, and intellectual virtues
- `docs/sprach-welten/` — 8 numbered modules on language, society, media, and philosophy

Sidebars are **auto-generated** from directory structure (`sidebars.ts` uses `autogenerated`). No manual sidebar maintenance is needed. Files and folders prefixed with `_` are excluded from the build entirely — use this for drafts and work-in-progress.

### Localization

English translations mirror the German structure under `i18n/en/docusaurus-plugin-content-docs/current/`. To add English translations, copy the German file to the corresponding path under `i18n/en/`. Run `yarn write-translations:en` to regenerate the translation key scaffolding if UI strings need translating.

### Custom Components

React components in `src/components/` are used in MDX pages:
- `Tooltip/` — term tooltips backed by `data/glossary.json`
- `IntroTopicBox/` — topic intro boxes used on landing pages
- `HomepageFeatures/` — homepage feature cards
- `BrowserWindow/` — iframe embed component

### Notion Integration

`tools/sync_kritisches_denken_notion.js` syncs content bi-directionally with Notion. Sync metadata lives in `sync_data/`. The VS Code MCP config (`.vscode/mcp.json`) wires the Notion MCP server for AI-assisted editing.

### AI Prompts

`.continue/prompts.md` and `.github/prompts/` contain reusable prompts for translating German docs to English, generating new topic pages, and explaining content — useful context when doing content work.

## Key Configuration

- `docusaurus.config.ts` — site URL, locales, plugins (Lunr search, image zoom, KaTeX, Mermaid), navbar/footer
- `.markdownlint.json` — markdown linting rules applied to all `.md`/`.mdx` files
- `data/glossary.json` — glossary entries powering the `Tooltip` component

## Content Conventions

- Blog post directories: `blog/YYYY-MM-DD-slug/`
- Doc frontmatter uses `sidebar_position`, `title`, `description`, and `tags` fields
- Math is supported via KaTeX (`$inline$` and `$$block$$`)
- Mermaid diagrams are enabled (fenced code blocks with `mermaid` language tag)
- File naming uses dashes, not underscores (recent migration ongoing)
