## Installation

```bash
yarn
```

## Local Development

```bash
yarn start

yarn run start --locale en
```

This command starts a local development server and opens up a browser window. Most changes are reflected live without having to restart the server.

## Build

```bash
yarn build
```

This command generates static content into the `build` directory and can be served using any static contents hosting service.

## Clear build cache

```bash
rm -rf .docusaurus build
```

or you do a clear

```
docusaurus clear
```

## Online Editing with Sveltia

[Sveltia CMS](https://github.com/sveltia/sveltia-cms) is a Git-backed CMS (drop-in replacement for Decap/Netlify CMS) that lets editors create and update Markdown content directly on GitHub through a browser UI — no local checkout, no `yarn`, no terminal.

### Access

- Production: <https://sprachlichtung.org/admin/>
- Local dev: <http://localhost:3000/admin/>

Editors sign in with their GitHub account. They must have write access to the [hecmec/sprachlichtung](https://github.com/hecmec/sprachlichtung) repository — Sveltia commits changes directly to the `main` branch on their behalf.

### How it's wired up

Two files in [static/admin/](static/admin/) drive everything:

- [static/admin/index.html](static/admin/index.html) — loads the Sveltia CMS bundle from the unpkg CDN.
- [static/admin/config.yml](static/admin/config.yml) — backend, media paths, i18n locales, and collection schemas.

Because the folder lives under `static/`, Docusaurus copies it verbatim into the build output, so the CMS is served at `/admin/` on the deployed site.

### Backend & auth

```yaml
backend:
  name: github
  repo: hecmec/sprachlichtung
  branch: main
```

The `github` backend uses GitHub's device-flow OAuth via Sveltia's hosted auth service — no Netlify Identity, no self-hosted proxy, no client secret to manage. The first sign-in pops up a GitHub authorization prompt; after that Sveltia stores the token in the browser.

### Media uploads

```yaml
media_folder: static/img/uploads
public_folder: /sprachlichtung/img/uploads
```

Images dropped into the CMS land in [static/img/uploads/](static/img/uploads/) and are referenced with the `/sprachlichtung/` baseUrl prefix so links resolve correctly on GitHub Pages.

### Internationalization

```yaml
i18n:
  structure: multiple_folders
  locales: [de, en]
  default_locale: de
```

German entries are saved under `docs/<collection>/`; English entries are mirrored under `i18n/en/docusaurus-plugin-content-docs/current/<collection>/`, matching Docusaurus's i18n layout.

### Collections

Two collections, one per top-level docs tree:

| Collection | Folder | Label |
| --- | --- | --- |
| `kritisches-denken` | [docs/kritisches-denken/](docs/kritisches-denken/) | Kritisches Denken |
| `sprach-welten` | [docs/sprach-welten/](docs/sprach-welten/) | Sprach-Welten |

Each entry exposes the standard Docusaurus frontmatter fields (`title`, `description`, `sidebar_position`, `tags`) plus a Markdown body editor. `create: true` lets editors add new pages from the UI.

### Adding a collection

1. Append a new entry to `collections:` in [static/admin/config.yml](static/admin/config.yml) pointing at a folder under `docs/`.
2. Mirror the field list from an existing collection unless the schema needs to differ.
3. Commit and push — the change is live on the next deploy.

### Limitations to know about

- Sidebars are autogenerated from the filesystem (see [sidebars.ts](sidebars.ts)), so newly created pages appear in the sidebar automatically after the next build.
- Files prefixed with `_` are excluded from the Docusaurus build but **are still visible** in the Sveltia UI — treat them as drafts.
- Sveltia commits straight to `main`; there is no review/PR workflow configured. If that becomes necessary, set `publish_mode: editorial_workflow` in `config.yml`.
